
All Ergoplank family keyboards use a symmetric stagger of 0.25u between the four non-bottom rows of the keyboard,
except that many models use a 0.5 stagger for the lowest row, because that leads to fewer gaps in the keyboard (and it's the conventional lower-row stagger since the invention of the typewriter).

Most designs use a symmetric hand position except some Ergoboards have a little large edge on the right to accomodate the cursor keys there. This is why the Ergoboard has so many variants, but at least all have the same total width and variants with the same hand distance have the same staggering shape and mostly the same key sizes. 
(The exceptions are keys not filling their gaps for design reasons.)

Ergoboard:
 - The 16/4.5 Low- vs Mid-shift variants differ only in the frame mapping and size of Shift and Delete keys. 
 - The same is true for the 16/4 Low- vs Mid-shift variants, which are four in total.


The Ergoboard and Thumbs Up are in the comfortable position to have the Return key in or near its conventional spot on the right edge.
This makes them relatively inert towards both a Mid-shift modification (Return simply moves up or down if it's in the way)
and a bottom-row modification (Return is simply not concerned).
The Thumbs Up was specifically designed to sacrifice some ergonomics (like wide hand position and symmetric staggering) to preserve
some of the looks of traditional keyboards. 
The Ergoboard, on the other hand, has the hard-to-use space above the arrow cluster, where there's naturally a larger key.
In any case, Ergoboard and Thumbs Up also manage to nicely uphold the design rule that all character keys are 1u wide!


Ergoslat and Ergoplank are very different in this regard, because they push the hand positions as far out as possible (but still leaving one column outside the pinkies).
This leaves only a 1u key in the standard Return key position, which is why we moved Return to the "second space bar" by default.
This leads to a very aesthetic look, but becomes weird when a letter key is mapped to one of the two thumb keys opposite the actual space bar.
Of course, we could replace another key from the bottom row than Return. 
Especially in the Mid-shift variant, any modifier would be well at home on the right end bottom row, where there's the widest key on the board.
However, I decided to keep the key assignment of the lower row independent of thumb key usage: there's always an active "angle-mod", 
meaning the below-pinky key is a 1u character key. 
Its advantage is that both sides are symmetrical in the key size and that characters map to 1u keys, which is always nice, and we don't always get done. 
Maybe at some point far in the future, we can add an explicit variant to map Return to this key.
This sounds like a super fun thing to do. 

Three options:
 - Low-shift (traditional)
 - Mid-shift with angle mod (ergonomic)
 - Mid-shift with low Ctrl left and Return right (aesthetic)
   This latter would be much like my personal split ortho layout.
